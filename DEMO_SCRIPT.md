# 3 分钟 Demo 讲解稿

## 0:00-0:20 定位

这个项目不是行程生成器，而是“本地闲时消费执行 Agent”。用户只说一句话，系统会把模糊需求拆成时间、同行人、预算、距离、餐饮和体力约束，然后生成可确认、可下单、可兜底的美团服务包。

## 0:20-1:30 亲子主线

输入：周六下午想和老婆孩子出去玩 4 小时，孩子 5 岁，别离家太远，老婆最近在减肥。

展示顺序：

1. 左侧理解结果：识别亲子家庭、3 人、4 小时时间窗、距离不要太远、低负担餐饮。
2. 中间 Agent Loop Trace：Planner 拆解需求；Researchers 以并行泳道查天气/路线、活动/票务、餐厅/订座、团购/加购、通知/提醒。
3. 候选方案：展示推荐方案、时间轴、预算、评分维度和推荐理由。
4. 右侧服务包：展示团购套餐、优惠、错峰策略和执行队列。

讲解重点：评委看到的不是大模型直接吐文本，而是先查证再合成，再把方案变成动作队列。

## 1:30-2:20 异常自修复

点击“景点无票”。

讲解：系统模拟原活动无票/限流，Trace 中活动票务泳道标记兜底，方案卡显示“活动已替换”，执行队列出现“重排校验”。这说明异常不是写在文档里，而是在产品里显性可见。

可补充点击“人数变化”。

讲解：人数临时增加后，系统按新人数重新检查票量、餐厅桌型和预算，避免把不可执行方案伪造成成功。

## 2:20-3:00 确认执行与边界

点击“确认并执行选中方案”。

展示：活动票、餐厅订座、排队号、团购券、加购、分享卡片和出发提醒按队列模拟执行。分享卡片会生成可发给老婆或朋友群的摘要。

边界说明：本 Demo 不接真实美团、真实地图、真实餐厅库存、真实支付、真实预约或真实消息系统；所有外部能力都是本地 Mock，用于展示 Agent 的规划、查证、确认、执行和兜底闭环。
